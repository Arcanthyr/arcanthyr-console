WORKFLOW REMINDER:
==================

LOCAL DEVELOPMENT:
1. Start Wrangler: wrangler dev
2. Edit worker.js or app.js
3. Save → auto-reload/refresh browser
4. Test at http://127.0.0.1:8787

DEPLOY TO PRODUCTION:
1. Change API URLs in app.js back to production URLs (above)
2. Save app.js
3. Commit and push to GitHub (triggers deployment)
4. Test at https://www.arcanthyr.com

LOCAL DATABASE:
- Stored in: .wrangler/state/v3/d1
- Separate from production data
- Can reset anytime with: wrangler d1 execute arcanthyr --local --file=schema.sql

PRODUCTION DATABASE:
- Accessed via: wrangler d1 execute arcanthyr (without --local flag)
- Contains your real data
- Export with: wrangler d1 export
